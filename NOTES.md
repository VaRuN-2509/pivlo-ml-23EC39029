My best configuration uses a custom Byte-Pair Encoding (BPE) tokenizer (vocab size 2048) to drastically compress the 3-byte Devanagari characters in the Hindi corpus. This allows the model to capture far more semantic meaning within its context window without wasting steps predicting deterministic UTF-8 continuation bytes. To maximize the strict 2,000,000 parameter budget, I tied the input embedding and output head weights. I also modernized the architecture by swapping standard Multi-Head Attention for Multi-Query Attention (MQA) and replacing absolute positional embeddings with cached Rotary Positional Embeddings (RoPE). These parameter-saving changes allowed me to widen the network's embedding dimension to 216 while remaining safely under the 2M cap. The network also utilizes SwiGLU activations and RMSNorm, yielding better parameter efficiency and faster CPU computation. For the training loop, I upgraded to AdamW with gradient clipping (1.0) and applied a cosine learning rate schedule with a 200-step linear warmup. Finally, I utilized gradient accumulation (4 micro-batches of size 16) to simulate an effective batch size of 64, maximizing learning efficiency per step without crashing the laptop's memory and yielding the lowest possible Bits Per Byte (BPB) score.
